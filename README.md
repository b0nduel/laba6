# DCMotor_Project

Проект MATLAB/Simulink для моделирования и тестирования системы управления двигателем постоянного тока (ДПТ).

## Цель проекта
Целью проекта является создание структурированной и воспроизводимой среды моделирования ДПТ с автоматической инициализацией параметров, тестированием моделей и очисткой рабочего пространства.

## Задачи
1. Организовать файлы проекта в понятную иерархию.
2. Автоматизировать загрузку и очистку переменных.
3. Реализовать тесты для проверки работоспособности модели.
4. Обеспечить воспроизводимость результатов и удобство совместной работы.

│ └── shortcuts/
│ └──── clear_constants.m # Shortcut для быстрой очистки переменных
├── tests/
│ ├── impulse_response.slx # Тест на импульсное воздействие (подсистема)
│ ├── step_response.slx # Тест на ступенчатое воздействие (полная система)

## Описание файлов

### Модели (`models/`)
- **Motor.slx** — основная Simulink-модель двигателя постоянного тока. Содержит расчётные блоки и подсистемы, реализующие математическую модель ДПТ.

### Скрипты (`scripts/`)
#### Инициализация (`initialization/`)
- **load_constants.m** — загружает в рабочую область параметры двигателя:
  - `Pn`, `Nn`, `Un`, `In` — номинальные значения мощности, скорости, напряжения и тока.
  - `Ra`, `La` — сопротивление и индуктивность якоря.
  - `Km`, `Kw` — конструктивные коэффициенты момента и скорости.
  - `Te` — электрическая постоянная времени.
  - `Mf`, `Ml` — моменты трения и нагрузки.

#### Деинициализация (`deinitialization/`)
- **clear_trash.m** — удаляет сгенерированные файлы компиляции моделей (`.slxc`, `.elf`, `.zip`) и папку `slprj`.
- **unload_workspace.m** — очищает переменные проекта (`DCMotor_Project`, `out`) из Workspace.

#### Shortcuts (`shortcuts/`)
- **clear_constants.m** — shortcut для быстрой очистки всех переменных в Workspace.

### Тесты (`tests/`)
- **step_response.slx** — тест полной системы на ступенчатое воздействие. Проверяет время переходного процесса и перерегулирование.
- **impulse_response.slx** — тест подсистемы на импульсное (дельта) воздействие.
- **impulse_response_HarnessInputs.mat** — файл с входными данными для импульсного теста.

### Прочие файлы
- **DCMotor_Project.prj** — проект MATLAB, обеспечивающий автоматическую инициализацию и деинициализацию.
- **README.md** — документация проекта.

## Порядок работы с проектом

1. Откройте проект через `DCMotor_Project.prj` (File → Open Project).
   - Автоматически выполнится `load_constants.m`, переменные появятся в Workspace.
2. Для работы с моделью откройте `models/Motor.slx`.
3. Для запуска тестов откройте файлы в папке `tests/` и выполните симуляцию.
4. Для очистки переменных используйте shortcut `clear_constants` (в панели Shortcuts проекта).
5. При закрытии проекта автоматически выполнятся скрипты деинициализации:
   - `clear_trash.m` — удалит временные файлы.
   - `unload_workspace.m` — очистит Workspace.

## Особенности запуска
- Проект настроен так, чтобы Workspace до и после работы с проектом оставался неизменным.
- Все пути в скриптах заданы относительно корня проекта.
- При работе с тестами убедитесь, что модель `Motor.slx` загружена и доступна.

## Тестирование
Проект включает два теста:
1. **Ступенчатое воздействие** — проверка времени переходного процесса (0.5с<) и перерегулирования (≈30%).
2. **Импульсное воздействие** — проверка реакции подсистемы на дельта-функцию.

Тесты расположены в `tests/` и могут быть запущены непосредственно из Simulink.
